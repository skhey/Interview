## Understanding Backpressure

Backpressure is a **flow control mechanism** that prevents a fast data producer from overwhelming a slow consumer. When the receiving end processes data slower than the sending end pushes it, data accumulates behind a buffer — like a clog in a pipe. Node.js streams have built-in backpressure handling that temporarily halts the data source until the consumer catches up, preventing memory from growing uncontrollably and eventually crashing the process.

**Without backpressure**, an application reading a large file and processing it slowly would buffer the entire file in memory, leading to memory exhaustion. **With backpressure**, the readable stream is automatically paused when the writable stream's internal buffer fills up, then resumed when buffer space becomes available.

## The Four Fundamental Stream Types

Node.js provides four base stream classes, all inheriting from `EventEmitter`:

### 1. Readable Streams

A **Readable stream** produces data that can be consumed. Data flows **out** of a Readable.

**Key properties and methods:**
- **Events**: `data` (chunk available), `end` (no more data), `error` (stream error), `readable` (data can be read)
- **Modes**: **Flowing mode** (data automatically emitted via `data` event) vs **Paused mode** (explicit `read()` calls required)
- **highWaterMark**: Internal buffer size threshold (default 16 KB for buffers, 16 objects for object mode)

**Built-in Readable streams:**
- `fs.createReadStream()` – reading files
- `http.IncomingMessage` – incoming HTTP request body
- `process.stdin` – standard input

```javascript
const fs = require('node:fs');

const readable = fs.createReadStream('/var/log/app.log', {
  encoding: 'utf8',
  highWaterMark: 64 * 1024  // 64 KB chunks
});

readable.on('data', (chunk) => {
  console.log(`Received ${chunk.length} characters`);
});

readable.on('end', () => {
  console.log('Finished reading');
});

readable.on('error', (err) => {
  console.error('Read error:', err.message);
});
```

### 2. Writable Streams

A **Writable stream** consumes data. Data flows **into** a Writable.

**Key properties and methods:**
- **`write(chunk)`**: Writes data; returns `true` if buffer has room, `false` if buffer is full
- **`end([chunk])`**: Signals no more data; optionally writes final chunk
- **Events**: `drain` (buffer emptied, ready for more), `finish` (all data flushed), `error`
- **highWaterMark**: Buffer size before `write()` returns `false`

**Built-in Writable streams:**
- `fs.createWriteStream()` – writing to files
- `http.ServerResponse` – outgoing HTTP response
- `process.stdout` / `process.stderr` – standard output/error

```javascript
const fs = require('node:fs');

const writable = fs.createWriteStream('./output.log', {
  encoding: 'utf8',
  highWaterMark: 64 * 1024
});

// write() returns false when buffer is full
const canContinue = writable.write('First line\n');
console.log('Buffer has room:', canContinue);

writable.end('Final line\n', () => {
  console.log('All data flushed to disk');
});
```

### 3. Duplex Streams

A **Duplex stream** implements both Readable and Writable interfaces. The two sides are **independent** — data written to the Writable side does not automatically appear on the Readable side; you must implement that relationship.

**Built-in Duplex streams:**
- `net.Socket` – TCP sockets
- WebSocket connections
- `stream.Duplex` – base class for custom implementations

```javascript
const net = require('node:net');

const server = net.createServer((socket) => {
  // socket is a Duplex stream
  socket.on('data', (data) => {
    // Echo data back, uppercased
    socket.write(data.toString().toUpperCase());
  });

  socket.on('end', () => {
    console.log('Client disconnected');
  });
});

server.listen(3000, () => {
  console.log('Echo server on port 3000');
});
```

### 4. Transform Streams

A **Transform stream** is a special Duplex where the **output is derived from the input**. Data goes through a transformation function that modifies it as it passes through.

Transform streams have **two internal buffers**: a write buffer (incoming data) and a read buffer (transformed output). Data from the write buffer is passed to `_transform()`, and the implementation must push transformed data to the read buffer.

**Built-in Transform streams:**
- `zlib.createGzip()` / `createGunzip()` – compression
- `crypto.createCipheriv()` / `createDecipheriv()` – encryption
- `stream.Transform` – base class for custom transforms

```javascript
const { Transform } = require('node:stream');

// Custom Transform: uppercase converter
const uppercaseTransform = new Transform({
  transform(chunk, encoding, callback) {
    // Convert chunk to uppercase and push to readable side
    this.push(chunk.toString().toUpperCase());
    callback();
  }
});

// CSV to JSON line-by-line
const csvToJson = new Transform({
  transform(chunk, encoding, callback) {
    const line = chunk.toString().trim();
    if (!line) return callback();
    
    const [name, age, email] = line.split(',');
    this.push(JSON.stringify({ name, age: parseInt(age), email }) + '\n');
    callback();
  }
});

// Usage
fs.createReadStream('data.csv')
  .pipe(csvToJson)
  .pipe(fs.createWriteStream('data.json'));
```

**Critical: Handling Transform Backpressure**

A common mistake is calling `callback()` before the transformed data has been fully consumed downstream. When output data volume exceeds input volume (e.g., expanding compressed data), the Readable buffer can overflow. Always respect backpressure by checking `push()` return value:

```javascript
const { Transform } = require('node:stream');

class SafeTransform extends Transform {
  _transform(chunk, encoding, callback) {
    // Process chunk...
    const output = this.processChunk(chunk);
    
    // push() returns false if buffer is full — respect it
    if (!this.push(output)) {
      // Buffer full; pause until drain
      this.once('drain', callback);
    } else {
      callback();
    }
  }
}
```

## Pipe Chaining and pipeline()

### `.pipe()` — Simple but Limited

The `.pipe()` method connects streams together, automatically managing backpressure:

```javascript
const fs = require('node:fs');
const zlib = require('node:zlib');

// Chain: read → compress → write
fs.createReadStream('large-file.txt')
  .pipe(zlib.createGzip())
  .pipe(fs.createWriteStream('large-file.txt.gz'));
```

**Limitation of `.pipe()`**: When an error occurs in any stream, `.pipe()` **does not automatically clean up** other streams. This causes:
- Resource leaks (open file descriptors)
- Inconsistent state (some streams continue processing)
- Silent failures (errors may not bubble up)

### `pipeline()` — Production-Ready Alternative

`stream.pipeline()` provides **automatic error propagation and guaranteed cleanup** — all streams are properly destroyed regardless of where an error occurs:

```javascript
const fs = require('node:fs');
const zlib = require('node:zlib');
const { pipeline } = require('node:stream/promises');

async function compressFile(inputPath, outputPath) {
  try {
    await pipeline(
      fs.createReadStream(inputPath),
      zlib.createGzip(),
      fs.createWriteStream(outputPath)
    );
    console.log('Compression succeeded');
  } catch (err) {
    console.error('Pipeline failed:', err);
  }
}
```

## Handling Slow Consumers (Backpressure in Practice)

When a writable stream cannot keep up with incoming data, the proper pattern is:

```javascript
const fs = require('node:fs');

function writeDataWithBackpressure(writable, data) {
  let i = 0;
  
  function write() {
    let ok = true;
    while (i < data.length && ok) {
      ok = writable.write(data[i]);
      i++;
    }
    
    if (i < data.length) {
      // Buffer full — wait for drain
      writable.once('drain', write);
    } else {
      writable.end();
    }
  }
  
  write();
}

const writable = fs.createWriteStream('output.txt');
const lines = Array.from({ length: 1e6 }, (_, i) => `Line ${i}\n`);
writeDataWithBackpressure(writable, lines);
```

```javascript
const { pipeline } = require('node:stream/promises');

// Backpressure handled automatically:
```

**HTTP Response Backpressure Example:**

```javascript
const http = require('node:http');
const fs = require('node:fs');

http.createServer((req, res) => {
  const stream = fs.createReadStream('large-video.mp4');
  
  stream.on('data', (chunk) => {
    // res.write() returns false when TCP buffer is full (~16KB)
    const canContinue = res.write(chunk);
    
    if (!canContinue) {
      // Slow client — pause reading until socket drains
      stream.pause();
      res.once('drain', () => stream.resume());
    }
  });
  
  stream.on('end', () => res.end());
}).listen(3000);
```

## Stream Buffering Explained

Each stream maintains an internal buffer controlled by `highWaterMark`:

- **Readable stream**: Buffer fills via `push()`; when buffer length ≥ highWaterMark, `push()` returns `false`, signaling to pause the underlying source
- **Writable stream**: Buffer fills via `write()`; when buffer length ≥ highWaterMark, `write()` returns `false`, triggering `drain` event when buffer empties

```
                    Readable Stream Buffer        Writable Stream Buffer
    Source ──push()──▶ [chunk][chunk][chunk] ──write()──▶ [chunk][chunk] ──▶ Destination
                              │                                    │
                         highWaterMark                       highWaterMark
                      (default 16 KB)                     (default 16 KB)
```

**Customizing buffer size:**

```javascript
// Larger buffer = fewer I/O operations, more memory
const readable = fs.createReadStream('file.txt', { highWaterMark: 1024 * 1024 }); // 1 MB

// Smaller buffer = lower memory, more frequent operations
const writable = fs.createWriteStream('file.txt', { highWaterMark: 4 * 1024 }); // 4 KB
```

## File Uploads and Downloads with Streams

### 📤 File Download (Server → Client)

Streaming prevents loading the entire file into memory, allowing files of any size to be served efficiently:

```javascript
const express = require('express');
const fs = require('node:fs');
const path = require('node:path');

const app = express();

app.get('/download/:filename', (req, res) => {
  const filename = req.params.filename;
  const filepath = path.join(__dirname, 'files', filename);
  
  // Validate and sanitize filename (prevent path traversal)
  if (!fs.existsSync(filepath)) {
    return res.status(404).json({ error: 'File not found' });
  }
  
  const stat = fs.statSync(filepath);
  
  // Set appropriate headers
  res.setHeader('Content-Disposition', `attachment; filename="${filename}"`);
  res.setHeader('Content-Type', 'application/octet-stream');
  res.setHeader('Content-Length', stat.size);
  
  // Stream file directly to response
  const readStream = fs.createReadStream(filepath);
  
  readStream.on('error', (error) => {
    console.error('Stream error:', error);
    if (!res.headersSent) {
      res.status(500).json({ error: 'Error streaming file' });
    }
  });
  
  readStream.pipe(res);
});
```

### 📥 File Upload (Client → Server)

```javascript
const express = require('express');
const fs = require('node:fs');
const multer = require('multer');

const app = express();
const upload = multer({ dest: 'uploads/' });

app.post('/upload', upload.single('file'), (req, res) => {
  // req.file contains metadata; file already saved by multer
  res.json({ message: 'File uploaded', filename: req.file.filename });
});

// Streaming upload without multer (direct pipe)
app.post('/upload-stream', (req, res) => {
  const writeStream = fs.createWriteStream(`uploads/${Date.now()}-file`);
  
  req.on('error', (err) => {
    console.error('Request error:', err);
    writeStream.destroy();
  });
  
  writeStream.on('finish', () => {
    res.json({ message: 'Upload complete' });
  });
  
  writeStream.on('error', (err) => {
    console.error('Write error:', err);
    res.status(500).json({ error: 'Upload failed' });
  });
  
  // Pipe request body directly to file
  req.pipe(writeStream);
});
```

## Real-World Applications

### 🎬 Video Streaming

```javascript
const express = require('express');
const fs = require('node:fs');

const app = express();

app.get('/video/:filename', (req, res) => {
  const filepath = `videos/${req.params.filename}`;
  const stat = fs.statSync(filepath);
  const fileSize = stat.size;
  const range = req.headers.range;
  
  if (range) {
    // Parse range header for seeking/scrubbing
    const parts = range.replace(/bytes=/, '').split('-');
    const start = parseInt(parts[0], 10);
    const end = parts[1] ? parseInt(parts[1], 10) : fileSize - 1;
    const chunkSize = (end - start) + 1;
    
    res.writeHead(206, {
      'Content-Range': `bytes ${start}-${end}/${fileSize}`,
      'Accept-Ranges': 'bytes',
      'Content-Length': chunkSize,
      'Content-Type': 'video/mp4'
    });
    
    fs.createReadStream(filepath, { start, end }).pipe(res);
  } else {
    res.writeHead(200, {
      'Content-Length': fileSize,
      'Content-Type': 'video/mp4'
    });
    fs.createReadStream(filepath).pipe(res);
  }
});
```

### 📊 Log Processing Pipeline (ETL)

```javascript
const fs = require('node:fs');
const { Transform, pipeline } = require('node:stream');
const readline = require('node:readline');

// Transform: Extract ERROR lines
const errorFilter = new Transform({
  transform(chunk, encoding, callback) {
    const lines = chunk.toString().split('\n');
    const errors = lines
      .filter(line => line.includes('[ERROR]'))
      .map(line => {
        // Parse and enrich
        const timestamp = line.substring(1, 25);
        const message = line.substring(line.indexOf('[ERROR]') + 8);
        return JSON.stringify({ timestamp, level: 'error', message }) + '\n';
      })
      .join('');
    
    this.push(errors);
    callback();
  }
});

// Pipeline: read → filter → write
async function processLogs() {
  await pipeline(
    fs.createReadStream('/var/log/app.log', { highWaterMark: 1024 * 1024 }),
    errorFilter,
    fs.createWriteStream('./errors.json')
  );
  console.log('Log processing complete');
}
```

### 🔐 Real-Time Encryption Proxy

```javascript
const net = require('node:net');
const crypto = require('node:crypto');
const { pipeline } = require('node:stream/promises');

// AES-256-GCM encryption/decryption streams
const algorithm = 'aes-256-gcm';
const key = crypto.randomBytes(32);
const iv = crypto.randomBytes(12);

const encryptStream = crypto.createCipheriv(algorithm, key, iv);
const decryptStream = crypto.createDecipheriv(algorithm, key, iv);

// Proxy server that encrypts all traffic
net.createServer((clientSocket) => {
  const serverSocket = net.createConnection({ port: 8080, host: 'backend' });
  
  // Client → Encrypt → Backend
  pipeline(clientSocket, encryptStream, serverSocket).catch(console.error);
  
  // Backend → Decrypt → Client
  pipeline(serverSocket, decryptStream, clientSocket).catch(console.error);
}).listen(9000);
```

## Summary Table

| Stream Type | Data Flow | Examples | Key Methods |
|-------------|-----------|----------|-------------|
| **Readable** | Produces data (out) | `createReadStream`, HTTP request | `pipe()`, `on('data')`, `read()` |
| **Writable** | Consumes data (in) | `createWriteStream`, HTTP response | `write()`, `end()`, `on('drain')` |
| **Duplex** | Both (independent) | TCP socket, WebSocket | Readable + Writable methods |
| **Transform** | Both (output from input) | Gzip, Crypto, custom transforms | `_transform()`, `_flush()` |

**Key takeaways:**
1. **Always prefer `pipeline()` over `.pipe()`** — automatic error handling and cleanup prevent resource leaks
2. **Respect `write()` return value** — when it returns `false`, wait for the `drain` event
3. **Transform streams need careful backpressure handling** — check `push()` return value and pause when buffer full
4. **Set appropriate `highWaterMark`** — balance memory usage vs. I/O frequency
5. **Streams are essential for large files** — processing multi-gigabyte files with streams keeps memory flat
