# Node.js Child Processes & Worker Threads – Complete Guide

## Why Offload CPU-Intensive Tasks?

Node.js's single-threaded event loop is excellent for I/O-bound operations but struggles with CPU-intensive work. When a long-running synchronous task runs on the main thread, the event loop is **blocked** — no new requests can be processed until it finishes.

**The Problem:**
```javascript
// ❌ This blocks the entire server for ~5 seconds
app.get('/hash', (req, res) => {
  const hash = crypto.pbkdf2Sync('password', 'salt', 1000000, 64, 'sha512');
  res.json({ hash: hash.toString('hex') });
});
// During those 5 seconds, ALL other requests wait in queue
```

**The Solution:** Offload heavy work to separate processes or threads.

| Approach | Best For | Isolation | Overhead |
|----------|----------|-----------|----------|
| **Child Processes** | External binaries, complete isolation | Separate memory space | Higher |
| **Worker Threads** | JavaScript/Node.js code, shared memory | Shared memory space | Lower |

## Child Processes (`child_process` Module)

Child processes run **external commands or scripts** in separate OS processes. Each has its own V8 instance and memory.

### The Four Ways to Create Child Processes

| Method | Purpose | Communication |
|--------|---------|---------------|
| `spawn()` | Launch a command with streaming I/O | stdout/stderr streams |
| `exec()` | Run command and buffer output | Callback with full output |
| `execFile()` | Execute a file directly (no shell) | Callback with output |
| `fork()` | Spawn a new Node.js process | IPC message channel |

### `spawn()` – Streaming Large Data

Best for **long-running processes** or **large data streams**.

```javascript
const { spawn } = require('node:child_process');

// Example: Convert video using ffmpeg
const ffmpeg = spawn('ffmpeg', [
  '-i', 'input.mp4',
  '-c:v', 'libx264',
  '-preset', 'fast',
  'output.mp4'
]);

ffmpeg.stdout.on('data', (data) => {
  console.log(`stdout: ${data}`);
});

ffmpeg.stderr.on('data', (data) => {
  console.error(`stderr: ${data}`);
});

ffmpeg.on('close', (code) => {
  console.log(`ffmpeg exited with code ${code}`);
});

// Pipe output to file
ffmpeg.stdout.pipe(fs.createWriteStream('progress.log'));
```

### `exec()` – Simple Commands with Buffered Output

Good for **short commands** with limited output (max 1MB by default).

```javascript
const { exec } = require('node:child_process');

exec('ls -la /tmp', { maxBuffer: 1024 * 1024 }, (error, stdout, stderr) => {
  if (error) {
    console.error(`Error: ${error.message}`);
    return;
  }
  if (stderr) {
    console.error(`stderr: ${stderr}`);
    return;
  }
  console.log(`stdout: ${stdout}`);
});
```

### `execFile()` – Execute Without Shell (More Secure)

```javascript
const { execFile } = require('node:child_process');

execFile('node', ['--version'], (error, stdout, stderr) => {
  if (error) throw error;
  console.log(`Node version: ${stdout}`);
});
```

### `fork()` – Node.js-Specific with IPC

Dedicated for spawning **Node.js modules** with a built-in IPC channel.

```javascript
// main.js
const { fork } = require('node:child_process');

const worker = fork('./worker.js');

worker.on('message', (msg) => {
  console.log('Message from worker:', msg);
});

worker.send({ task: 'compute', data: [1, 2, 3, 4, 5] });

// worker.js
process.on('message', (msg) => {
  if (msg.task === 'compute') {
    const result = msg.data.reduce((a, b) => a + b, 0);
    process.send({ result });
  }
});
```

## Worker Threads (`worker_threads` Module)

Worker threads run JavaScript **in parallel threads** within the same Node.js process, sharing memory (via `SharedArrayBuffer`) and communicating efficiently.

### Key Concepts

- Each worker has its **own event loop**
- Workers can share memory using `SharedArrayBuffer` and `Atomics`
- Communication via `postMessage()` (structured cloning)
- Lighter than child processes (no new V8 instance)

### Basic Worker Thread Example

```javascript
// main.js
const { Worker } = require('node:worker_threads');

function runHeavyTask(data) {
  return new Promise((resolve, reject) => {
    const worker = new Worker('./heavy-worker.js', {
      workerData: data  // Pass initial data
    });
    
    worker.on('message', resolve);
    worker.on('error', reject);
    worker.on('exit', (code) => {
      if (code !== 0) {
        reject(new Error(`Worker stopped with exit code ${code}`));
      }
    });
  });
}

// Usage
runHeavyTask({ numbers: Array.from({ length: 1e7 }, (_, i) => i) })
  .then(result => console.log('Sum:', result))
  .catch(err => console.error(err));
```

```javascript
// heavy-worker.js
const { parentPort, workerData } = require('node:worker_threads');

// CPU-intensive work
const sum = workerData.numbers.reduce((acc, val) => acc + val, 0);

// Send result back to main thread
parentPort.postMessage(sum);
```

### Worker Pool for Reusing Threads

Creating a new worker for each task has overhead. A **worker pool** reuses threads.

```javascript
// worker-pool.js
const { Worker } = require('node:worker_threads');
const os = require('node:os');

class WorkerPool {
  constructor(workerScript, poolSize = os.cpus().length) {
    this.workerScript = workerScript;
    this.poolSize = poolSize;
    this.workers = [];
    this.queue = [];
    this.activeWorkers = 0;
    this.nextWorkerId = 0;
  }
  
  async run(task) {
    return new Promise((resolve, reject) => {
      this.queue.push({ task, resolve, reject });
      this._processQueue();
    });
  }
  
  _processQueue() {
    if (this.queue.length === 0) return;
    
    if (this.activeWorkers < this.poolSize) {
      const { task, resolve, reject } = this.queue.shift();
      this._runTask(task, resolve, reject);
      this._processQueue(); // Try to start more tasks
    }
  }
  
  _runTask(task, resolve, reject) {
    this.activeWorkers++;
    
    const worker = new Worker(this.workerScript, {
      workerData: task
    });
    
    const workerId = ++this.nextWorkerId;
    console.log(`Worker ${workerId} started`);
    
    worker.on('message', (result) => {
      console.log(`Worker ${workerId} finished`);
      resolve(result);
      this.activeWorkers--;
      this._processQueue();
    });
    
    worker.on('error', (err) => {
      reject(err);
      this.activeWorkers--;
      this._processQueue();
    });
    
    worker.on('exit', (code) => {
      if (code !== 0) {
        reject(new Error(`Worker exited with code ${code}`));
        this.activeWorkers--;
        this._processQueue();
      }
    });
  }
}

module.exports = WorkerPool;
```

**Usage:**
```javascript
const WorkerPool = require('./worker-pool');

const pool = new WorkerPool('./hash-worker.js', 4);

async function hashPasswords(passwords) {
  const promises = passwords.map(pwd => pool.run({ password: pwd }));
  return Promise.all(promises);
}
```

## Real-World Use Cases

### 🔐 Password Hashing (bcrypt / pbkdf2)

Hashing is **intentionally slow** to prevent brute-force attacks. Doing it on the main thread blocks all requests.

```javascript
// hash-worker.js
const { parentPort, workerData } = require('node:worker_threads');
const crypto = require('node:crypto');

const { password, salt } = workerData;

// CPU-intensive hash (blocks only this worker thread)
const hash = crypto.pbkdf2Sync(password, salt, 100000, 64, 'sha512');

parentPort.postMessage(hash.toString('hex'));
```

```javascript
// main.js - Express route using worker pool
const WorkerPool = require('./worker-pool');
const pool = new WorkerPool('./hash-worker.js');

app.post('/hash', async (req, res) => {
  try {
    const { password } = req.body;
    const salt = crypto.randomBytes(16).toString('hex');
    
    // Offload to worker thread - main thread stays responsive
    const hash = await pool.run({ password, salt });
    
    res.json({ hash, salt });
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
});
```

### 🤖 Machine Learning Inference with TensorFlow.js

```javascript
// ml-worker.js
const { parentPort, workerData } = require('node:worker_threads');
const tf = require('@tensorflow/tfjs-node');

let model = null;

async function loadModel() {
  if (!model) {
    // Load once per worker (cached)

  return model;
}

async function predict(inputData) {
  const model = await loadModel();
  const tensor = tf.tensor(inputData);
  const prediction = model.predict(tensor);
  const result = await prediction.data();
  
  // Cleanup tensors
  tensor.dispose();
  prediction.dispose();
  
  return Array.from(result);
}

// Handle messages
parentPort.on('message', async (msg) => {
  if (msg.type === 'predict') {
    try {
      const result = await predict(msg.data);
      parentPort.postMessage({ id: msg.id, result });
    } catch (err) {
      parentPort.postMessage({ id: msg.id, error: err.message });
    }
  }
});
```

```javascript
// main.js - ML prediction pool
class MLWorkerPool {
  constructor(workerScript, poolSize) {
    this.workers = [];
    this.queue = [];
    this.requestId = 0;
    this.pendingRequests = new Map();
    
    for (let i = 0; i < poolSize; i++) {
      const worker = new Worker(workerScript);
      
      worker.on('message', (msg) => {
        const resolver = this.pendingRequests.get(msg.id);
        if (resolver) {
          if (msg.error) {
            resolver.reject(new Error(msg.error));
          } else {
            resolver.resolve(msg.result);
          }
          this.pendingRequests.delete(msg.id);
        }
        
        // Return worker to pool and process next
        this.idleWorkers.push(worker);
        this._processQueue();
      });
      
      this.idleWorkers = [];
      this.idleWorkers.push(worker);
    }
  }
  
  async predict(data) {
    return new Promise((resolve, reject) => {
      const id = ++this.requestId;
      this.pendingRequests.set(id, { resolve, reject });
      this.queue.push({ id, data });
      this._processQueue();
    });
  }
  
  _processQueue() {
    if (this.queue.length === 0 || this.idleWorkers.length === 0) return;
    
    const worker = this.idleWorkers.pop();
    const task = this.queue.shift();
    
    worker.postMessage({ type: 'predict', id: task.id, data: task.data });
  }
}

// Usage
const mlPool = new MLWorkerPool('./ml-worker.js', 4);

app.post('/predict', async (req, res) => {
  try {
    const input = req.body.image; // Preprocessed image data
    const prediction = await mlPool.predict(input);
    res.json({ prediction });
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
});
```

```javascript
// etl-pipeline.js
const { Worker } = require('node:worker_threads');
const fs = require('node:fs');
const readline = require('node:readline');
const { pipeline } = require('node:stream/promises');
const { Transform } = require('node:stream');

class ParallelTransform extends Transform {
  constructor(workerScript, concurrency = 4) {
    super({ objectMode: true });
    this.workerScript = workerScript;
    this.concurrency = concurrency;
    this.workers = [];
    this.queue = [];
    this.activeWorkers = 0;
    this.processing = false;
  }
  
  _transform(chunk, encoding, callback) {
    this.queue.push(chunk);
    this._processQueue(callback);
  }
  
  _flush(callback) {
    this.processing = true;
    const checkDone = () => {
      if (this.activeWorkers === 0 && this.queue.length === 0) {
        callback();
      } else {
        setTimeout(checkDone, 10);
      }
    };
    checkDone();
  }
  
  _processQueue(userCallback) {
    if (this.activeWorkers >= this.concurrency || this.queue.length === 0) {
      userCallback();
      return;
    }
    
    const chunk = this.queue.shift();
    this.activeWorkers++;
    
    const worker = new Worker(this.workerScript);
    
    worker.on('message', (result) => {
      result.forEach(item => this.push(item));
      this.activeWorkers--;
      this._processQueue(() => {});
    });
    
    worker.on('error', (err) => {
      this.emit('error', err);
      this.activeWorkers--;
    });
    
    worker.postMessage(chunk);
    userCallback();
  }
}

// Usage: Process large CSV in parallel
async function processLargeFile() {
  const readStream = fs.createReadStream('huge-data.csv');
  const writeStream = fs.createWriteStream('transformed-data.json');
  
  const rl = readline.createInterface({ input: readStream });
  
  let chunk = [];
  let chunkSize = 1000;
  
  await pipeline(
    rl,
    new Transform({
      objectMode: true,
      transform(line, enc, cb) {
        // Parse CSV line
        const [firstName, lastName, age] = line.split(',');
        chunk.push({ firstName, lastName, age: parseInt(age) });
        
        if (chunk.length >= chunkSize) {
          this.push(chunk);
          chunk = [];
        }
        cb();
      },
      flush(cb) {
        if (chunk.length) this.push(chunk);
        cb();
      }
    }),
    new ParallelTransform('./transform-worker.js', 8),
    new Transform({
      objectMode: true,
      transform(item, enc, cb) {
        cb(null, JSON.stringify(item) + '\n');
      }
    }),
    writeStream
  );
  
  console.log('Processing complete');
}
```

## Child Process vs Worker Thread: Decision Guide

| Criteria | Child Process | Worker Thread |
|----------|---------------|---------------|
| **Memory Isolation** | Complete (separate process) | Shared (can access same heap) |
| **Startup Time** | Slower (~30-50ms) | Faster (~5-10ms) |
| **Communication** | IPC (serialized) | postMessage or SharedArrayBuffer |
| **External Binaries** | ✅ Yes (ffmpeg, imagemagick) | ❌ No |
| **Native Modules** | ✅ Safe isolation | ⚠️ Some native modules may crash process |
| **CPU-bound JS** | ✅ Works | ✅ Best choice |
| **Resource Overhead** | Higher (separate V8) | Lower (shared V8 instance) |

### When to Use Each

**Use Child Processes (`spawn`/`exec`/`execFile`):**
- Running external commands (ffmpeg, ImageMagick, system utilities)
- Complete isolation needed (untrusted code, native modules that crash)
- Independent Node.js scripts with `fork()`

**Use Worker Threads:**
- CPU-intensive JavaScript (hashing, encryption, compression)
- Parallel data processing (arrays, objects)
- Tasks that benefit from shared memory
- High-frequency tasks (lower startup overhead)

## Message Passing Patterns

### Structured Clone Algorithm

Both `worker_threads.postMessage()` and `process.send()` use the **structured clone algorithm**:

```javascript
// Works: primitives, arrays, plain objects, Date, RegExp, Map, Set
worker.postMessage({
  string: 'hello',
  number: 42,
  date: new Date(),
  map: new Map([['key', 'value']])
});

// Does NOT work: functions, classes, DOM nodes, circular references
// worker.postMessage({ fn: () => {} }); // ❌ Throws DataCloneError
```

### Transferable Objects (Zero-Copy)

For large binary data (ArrayBuffer, MessagePort), use **transfer list** to avoid copying:

```javascript
const buffer = new ArrayBuffer(1024 * 1024 * 100); // 100 MB

// Without transfer: copies 100 MB (slow, double memory)
worker.postMessage({ buffer });

// With transfer: ownership moves (zero copy, original becomes unusable)
worker.postMessage({ buffer }, [buffer]);
// After transfer, buffer.byteLength === 0 in sender
```

### SharedArrayBuffer and Atomics (True Shared Memory)

```javascript
// main.js
const { Worker } = require('node:worker_threads');

// Create shared buffer (1024 integers)
const sharedBuffer = new SharedArrayBuffer(1024 * Int32Array.BYTES_PER_ELEMENT);
const sharedArray = new Int32Array(sharedBuffer);

// Spawn multiple workers that operate on same memory
const workers = [];
for (let i = 0; i < 4; i++) {
  workers.push(new Worker('./shared-worker.js', {
    workerData: { sharedBuffer, workerId: i }
  }));
}

// Wait for results
setTimeout(() => {
  console.log('Final array:', sharedArray);
}, 1000);
```

```javascript
// shared-worker.js
const { parentPort, workerData } = require('node:worker_threads');

const { sharedBuffer, workerId } = workerData;
const sharedArray = new Int32Array(sharedBuffer);

// Each worker increments a different section
const start = workerId * 256;
const end = start + 256;

for (let i = start; i < end; i++) {
  // Atomic operation prevents race conditions
  Atomics.add(sharedArray, i, 1);
  
  // Simulate work
  Atomics.wait(new Int32Array(sharedBuffer), 0, 0, 1);
}

parentPort.postMessage('done');
```

## Production Best Practices

### 1. Set Resource Limits

```javascript
const { Worker } = require('node:worker_threads');

const worker = new Worker('./worker.js', {
  resourceLimits: {
    maxOldGenerationSizeMb: 512,  // Max heap size
    maxYoungGenerationSizeMb: 64,
    codeRangeSizeMb: 32,
    stackSizeMb: 4
  }
});
```

### 2. Handle Uncaught Exceptions

```javascript
// In worker
process.on('uncaughtException', (err) => {
  console.error('Worker uncaught exception:', err);
  parentPort.postMessage({ error: err.message });
  process.exit(1);
});

process.on('unhandledRejection', (reason) => {
  console.error('Worker unhandled rejection:', reason);
  parentPort.postMessage({ error: String(reason) });
  process.exit(1);
});
```

### 3. Graceful Shutdown

```javascript
// main.js - Graceful shutdown with worker cleanup
const workers = new Set();

function shutdown() {
  console.log('Shutting down workers...');
  
  const promises = Array.from(workers).map(worker => {
    return new Promise((resolve) => {
      worker.postMessage({ type: 'shutdown' });
      worker.once('exit', resolve);
      setTimeout(() => {
        worker.terminate();
        resolve();
      }, 5000);
    });
  });
  
  Promise.all(promises).then(() => {
    console.log('All workers terminated');
    process.exit(0);
  });
}

process.on('SIGTERM', shutdown);
process.on('SIGINT', shutdown);
```

### 4. Monitor Worker Health

```javascript
class MonitoredWorker extends Worker {
  constructor(filename, options) {
    super(filename, options);
    this.startTime = Date.now();
    this.taskCount = 0;
    this.lastHeartbeat = Date.now();
    
    // Periodic heartbeat
    this.heartbeatInterval = setInterval(() => {
      this.postMessage({ type: 'heartbeat' });
    }, 5000);
    
    this.on('message', (msg) => {
      if (msg.type === 'heartbeat-ack') {
        this.lastHeartbeat = Date.now();
      }
    });
  }
  
  isHealthy() {
    // Worker unresponsive for >30 seconds
    return (Date.now() - this.lastHeartbeat) < 30000;
  }
  
  terminate() {
    clearInterval(this.heartbeatInterval);
    return super.terminate();
  }
}
```

### 5. Error Recovery with Circuit Breaker

```javascript
class WorkerCircuitBreaker {
  constructor(workerPool, options = {}) {
    this.pool = workerPool;
    this.failureThreshold = options.failureThreshold || 5;
    this.resetTimeout = options.resetTimeout || 60000;
    this.failureCount = 0;
    this.state = 'CLOSED'; // CLOSED, OPEN, HALF_OPEN
    this.nextAttempt = Date.now();
  }
  
  async execute(task) {
    if (this.state === 'OPEN') {
      if (Date.now() > this.nextAttempt) {
        this.state = 'HALF_OPEN';
      } else {
        throw new Error('Circuit breaker is OPEN');
      }
    }
    
    try {
      const result = await this.pool.run(task);
      
      if (this.state === 'HALF_OPEN') {
        this.state = 'CLOSED';
        this.failureCount = 0;
      }
      
      return result;
    } catch (err) {
      this.failureCount++;
      
      if (this.failureCount >= this.failureThreshold) {
        this.state = 'OPEN';
        this.nextAttempt = Date.now() + this.resetTimeout;
        console.error(`Circuit breaker OPEN until ${new Date(this.nextAttempt)}`);
      }
      
      throw err;
    }
  }
}
```

## Summary Table: Child Process vs Worker Thread

| Feature | `spawn`/`exec` | `fork` | Worker Thread |
|---------|---------------|--------|---------------|
| **Creates** | Any process | Node.js process | Thread in same process |
| **Memory** | Isolated | Isolated | Shared (optional) |
| **Communication** | stdout/stderr or IPC | IPC channel | postMessage / SharedArrayBuffer |
| **Startup Overhead** | High | Medium | Low |
| **Best For** | External tools | Separate Node.js apps | CPU-bound JS |
| **Data Transfer** | Serialized (slow) | Serialized | Transferable objects (fast) |

**Key Takeaways:**
1. **Use Worker Threads** for CPU-intensive JavaScript tasks — lowest overhead
2. **Use `spawn`** for external binaries like ffmpeg or ImageMagick
3. **Use `fork`** when you need a separate Node.js process with IPC
4. **Always pool** workers to avoid creation overhead
5. **Implement health checks** and graceful shutdown
6. **Monitor memory usage** — workers can leak just like the main thread
